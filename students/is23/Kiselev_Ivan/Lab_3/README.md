# Лабораторная работа 3: Простое веб-приложение. Верстка

**Цель:** Освоение базовых инструментов фронтенд-разработки: `Node.js`, `npm`, `Bootstrap`, `ES-модули`. Реализация простого двухстраничного веб-приложения с динамической генерацией контента на чистом JavaScript без фреймворков. Приложение реализует компонентный подход, работу с DOM, обработку событий и отображение уведомлений.

---

## Структура проекта

- **css/** — стили приложения:
  - `style.css` — кастомные стили для карточек, уведомлений и страниц
- **images/** — изображения для карточек:
  - `image1.jpg`, `image2.jpg`, `image3.jpg` — иллюстрации финансовых акций
- **components/** — переиспользуемые UI-компоненты:
  - `product-card/index.js` — карточка финансовой акции на главной странице (с кнопками «Подробнее», «Связаться», «Удалить»)
  - `product/index.js` — детальная карточка акции на странице продукта
  - `back-button/index.js` — кнопка возврата на главную страницу
  - `toast/index.js` — компонент всплывающих уведомлений (Toasts)
- **pages/** — логика страниц приложения:
  - `main/index.js` — главная страница: отображение списка акций, добавление/удаление карточек, обработка событий
  - `product/index.js` — страница детального просмотра выбранной акции
- **index.html** — точка входа. Подключает Bootstrap, кастомные стили и главный скрипт
- **main.js** — инициализация приложения, хранение состояния (список карточек), управление навигацией между страницами
- **package.json**, **package-lock.json** — метаданные и зависимости проекта (Bootstrap)
- **.gitignore** — исключение служебных файлов из Git (например, `node_modules/`)
- **README.md** — текущий документ

Проект использует **нативные ES-модули** (`import`/`export`) и **не требует сборки** (например, через Webpack или TypeScript). Весь код выполняется в браузере.

---

## Функциональность

- **Двустраничное приложение**: главная страница и страница продукта.
- **Динамическая генерация карточек** на основе данных (без хардкода в HTML).
- **Добавление карточек** по кругу: при нажатии на кнопку «+ Добавить акцию» последовательно добавляются акции 1 → 2 → 3 → 1 → ...
- **Удаление карточек** — каждая карточка имеет кнопку «Удалить» (работает только на главной странице).
- **Всплывающие уведомления (Toasts)** — появляются при нажатии на «Связаться»/«Запросить анализ»/«Активировать кэшбэк».
- **Переход между страницами** без потери состояния: добавленные/удалённые карточки сохраняются при переходе туда-сюда.
- **Сброс состояния при перезагрузке** — после обновления страницы отображаются только три исходные карточки.
- **Адаптивная вёрстка** с использованием Bootstrap 5.
- **Чистая архитектура**: разделение на страницы и компоненты, инкапсуляция логики.

## Как запустить

1. **Требования**:

   - Установленный `Node.js` (для работы с `npm`)
   - Расширение **Live Server** в VS Code (или любой локальный веб-сервер)
2. **Установка зависимостей**:

   ```bash
   npm install
   ```

   Эта команда установит Bootstrap в папку `node_modules/`.
3. **Подготовка изображений**:

   - Убедитесь, что в папке `images/` находятся файлы:
     - `image1.jpg`
     - `image2.jpg`
     - `image3.jpg`
   - Если изображений нет — можно временно заменить пути в `main.js` на плейсхолдеры-URL.
4. **Запуск**:

   - Откройте проект в VS Code.
   - ПКМ по файлу `index.html` → **Open with Live Server (или используйте любой другой способ запустить локальный сервер)**
   - Приложение откроется в браузере (обычно на `http://127.0.0.1:5500`).

> **Важно**: Приложение **не требует серверной части**. Вся логика выполняется в браузере.

## Технические особенности

- Все переходы между страницами реализованы **через перерисовку DOM**, а не через навигацию по разным HTML-файлам.
- Состояние приложения (список карточек) хранится в `main.js` и **сохраняется между переходами**, но **сбрасывается при перезагрузке** — как того требует условие.
- Для генерации уникальных идентификаторов карточек используется временная метка (`Date.now()`).
- Всплывающие уведомления реализованы вручную с использованием Bootstrap-стилей и автоматическим скрытием через 5 секунд.
