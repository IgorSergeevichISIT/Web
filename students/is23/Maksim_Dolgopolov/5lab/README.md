# Отчёт по лабораторной работе №5  
**Простое веб-приложение. Работа с API (Продолжение)**

---

## Цель работы

Целью работы было изучение современного подхода к выполнению HTTP-запросов с использованием `fetch` и асинхронного программирования через `async/await`. В рамках лабораторной работы требовалось переработать приложение, созданное в ЛР4, заменив устаревший `XMLHttpRequest` на современный `fetch`.

---

## Что сделано

Был переписан модуль `ajax.js`: вместо класса, основанного на `XMLHttpRequest`, реализован класс `Ajax` с асинхронным методом `post`, использующим `fetch`. Метод обрабатывает успешные ответы и ошибки, возвращает данные в формате JSON и передаёт их через callback. Добавлена обработка сетевых ошибок и ошибок HTTP.

В компонентах `MainPage` и `ProductPage` сохранена существующая логика работы с API VK, но теперь все запросы выполняются через обновлённый модуль `ajax`. Применён асинхронный подход: в `ajax.post` используется `await fetch`, что делает код более читаемым и устойчивым к ошибкам.

Вёрстка и функционал приложения остались без изменений: главная страница выводит карточки участников группы с фильтрацией, страница пользователя отображает детальную информацию. Все переходы, обработка кликов и отображение данных работают так же, как и в предыдущей версии, но с использованием современного API для сетевых запросов.

---

## Итог

В результате было создано приложение, полностью соответствующее требованиям задания: интеграция с VK API через `fetch`, асинхронная загрузка данных, фильтрация участников по типам и отображение детальных профилей. Код стал более чистым, поддерживаемым и соответствует современным стандартам JavaScript.
