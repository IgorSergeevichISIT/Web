# Отчёт по лабораторной работе №4  
**Простое веб-приложение. Работа с API**

---

## Цель работы

Целью работы было изучение взаимодействия с внешним API через XMLHttpRequest и создание веб-приложения, динамически загружающего и отображающего данные из VK API. В ходе выполнения предстояло реализовать двухстраничное приложение для просмотра участников группы ВКонтакте с возможностью фильтрации и детальной информацией о пользователях.

---

## Что сделано

Была создана модульная архитектура с разделением на слои: модули для работы с API (`ajax.js`, `urls.js`, `consts.js`), страницы (`MainPage`, `ProductPage`) и компоненты (`ProductCardComponent`, `ProductComponent`). Класс `Ajax` инкапсулирует логику POST-запросов через `XMLHttpRequest`, а класс `Urls` формирует конечные точки VK API с учётом токена доступа и версии.

На главной странице реализован динамический вывод карточек участников группы с использованием метода `groups.getMembers`. Добавлен компонент фильтрации по типу участников: «друзья», «не уверены», «руководители», «подписчики Donut». При выборе фильтра отправляется новый запрос с соответствующим параметром `filter`, после чего интерфейс обновляется.

При клике на карточку пользователя открывается страница с детальной информацией, загружаемой через метод `users.get`. Отображаются фото, имя, фамилия, статус, ID и время последнего посещения. Реализована кнопка «Назад» для возврата к списку.

Вёрстка выполнена с использованием Bootstrap, добавлены анимации фона и адаптивное расположение элементов. Обработаны случаи ошибок загрузки данных и пустых результатов.

---

## Итог

В результате разработано полнофункциональное веб-приложение, демонстрирующее работу с внешним REST API, динамическую загрузку данных, фильтрацию и навигацию между страницами. Приложение готово к подключению реальных данных VK после указания `groupId` и `accessToken` в файле конфигурации.
