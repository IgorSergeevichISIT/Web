# Лабораторная работа №4: Веб‑приложение «VK группа» (XMLHttpRequest)

**Студент:** Бородин Артём  
**Группа:** ИС‑23  

## 1. Краткое описание

В рамках лабораторной работы №4 реализовано одностраничное веб‑приложение, которое подключается к VK API и отображает список участников заданной группы во VK. Приложение показывает карточки пользователей с именем, фотографией и кнопкой перехода к подробной информации, а также позволяет открыть страницу профиля выбранного участника.

## 2. Используемые технологии

- JavaScript
- XMLHttpRequest для работы с VK API
- Модульная структура: `modules` (ajax, urls, consts) + `components` (UserCard, Filter, UserPage)

## 3. Функциональность приложения

В лабораторной работе №4 реализованы следующие возможности:

- **Загрузка списка участников** указанной VK‑группы через метод `groups.getMembers` и вывод их в виде карточек на главной странице.
- **Отображение карточек** с именем и аватаром пользователя; при клике по кнопке «Посмотреть профиль» открывается страница профиля.
- **Загрузка подробной информации** о пользователе (имя, фамилия, фото, город, дата рождения, ID).
- **Фильтрация списка** участников по категориям на главной странице (компонент Filter).
- **Разделение на слои**: модуль `ajax` инкапсулирует работу с XMLHttpRequest, модуль `urls` формирует ссылки к VK API, а компоненты отвечают за отрисовку интерфейса.
- **Кнопка возврата** на главную страницу из профиля пользователя.

## 4. Инструкция по запуску

### Требования
- Современный браузер (Chrome, Firefox)
- Доступ в интернет для подключения к VK API
- Расширение CORS в браузере

## 5. Использование приложения

- **Главная страница:** Просмотр списка участников группы с возможностью фильтрации
- **Фильтрация:** Введи текст в поле фильтра, чтобы найти участника по имени
- **Профиль:** Нажми кнопку «Посмотреть профиль» на карточке, чтобы открыть подробную информацию
- **Возврат:** Нажми кнопку «← Вернуться» чтобы вернуться на главную страницу

## 6. Технические особенности

### XMLHttpRequest vs Fetch
В этой лабораторной работе используется классический **XMLHttpRequest** для демонстрации традиционного подхода к асинхронным запросам. Более современный подход — использование **Fetch API** — будет рассмотрен в лабораторной работе №5.

### CORS и безопасность
- VK API требует корректного CORS‑заголовка при запросах с фронтенда
- Для локального развития используется CORS‑расширение браузера
- На продакшене необходимо использовать бэкенд‑прокси или серверное API

## 7. Заключение

Лабораторная работа №4 демонстрирует практическое применение XMLHttpRequest для работы с внешними API, разделение кода на модули и компоненты, а также основные принципы построения одностраничного веб‑приложения на JavaScript.
