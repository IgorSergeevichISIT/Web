## 1. Краткое описание

В рамках лабораторной работы №6 реализовано полноценное клиент‑серверное веб‑приложение для управления списком участников VK‑группы. Приложение построено на архитектуре **Backend For Frontend (BFF)**, где Node.js‑сервер выступает промежуточным слоем между клиентом и VK API. Фронтенд использует **Fetch API** вместо XMLHttpRequest для более современного подхода к асинхронным запросам.

## 2. Функциональность приложения

### Основные возможности

- **Загрузка участников** из VK API (на сервере) и кэширование в `users.json`
- **Отображение списка пользователей** с карточками (имя, фото, статус)
- **Добавление новых пользователей** через модальное окно (`AddUserModal`)
- **Редактирование информации** о пользователе (имя, статус, город, дата рождения)
- **Удаление пользователей** из списка
- **Просмотр профиля** пользователя на отдельной странице
- **Фильтрация списка** по имени и статусу
- **REST API эндпоинты:**
  - `GET /api/health` — проверка статуса сервера
  - `GET /api/users` — получить всех пользователей (с опциональной фильтрацией)
  - `GET /api/users/:id` — получить одного пользователя
  - `POST /api/users` — добавить нового пользователя
  - `PUT /api/users/:id` — обновить пользователя
  - `DELETE /api/users/:id` — удалить пользователя

### Дополнительное задание (Пункт 8)

Реализована архитектура **BFF**, которая обеспечивает:
- **Безопасность:** ACCESS_TOKEN хранится на сервере, не доступен браузеру
- **Трансформация данных:** Сервер формирует нужный формат данных для фронтенда
- **Кэширование:** Минимизация нагрузки на VK API через сохранение данных в `users.json`
- **Нормализация:** Изменения в VK API не влияют на фронтенд

## 3. Инструкция по запуску

### Требования
- Node.js (версия 14 или выше)
- npm (входит в Node.js)
- Доступ в интернет для подключения к VK API

### Установка и запуск

1. **Установите зависимости:**
   ```bash
   npm install
   ```

2. **Запустите сервер:**
   ```bash
   node server/server.js
   ```
   
   Или с использованием `nodemon` для автоперезагрузки при изменениях:
   ```bash
   npx nodemon server/server.js
   ```

3. **Откройте фронтенд в браузере:**
   ```
   http://localhost:3000
   ```

## 4. Использование приложения

### На главной странице
- Просмотр списка участников группы
- Фильтрация по имени через строку поиска
- Кнопка "Добавить пользователя" для открытия модального окна
- Нажмите на карточку пользователя для просмотра полного профиля

### В модальном окне добавления
- Заполните поля: имя, фамилия, статус, город, дата рождения
- Нажмите "Добавить" для сохранения нового пользователя

### На странице профиля
- Просмотр полной информации о пользователе
- Редактирование информации через формы
- Кнопка удаления пользователя
- Кнопка "Вернуться" для возврата на главную

### Backend For Frontend (BFF)
**Почему это нужно:**
- **Защита токена:** ACCESS_TOKEN хранится только на сервере, не в браузере
- **Трансформация:** Сервер оставляет только нужные поля из VK API
- **Кэширование:** Уменьшает нагрузку на VK API
