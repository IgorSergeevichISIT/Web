**Лабораторная работа №4: Веб‑приложение «VK группа» (XMLHttpRequest)**  

1. **Краткое описание**  
В ходе выполнения лабораторной работы №4 было разработано одностраничное веб‑приложение, взаимодействующее с VK API и отображающее перечень участников указанной группы VK. Приложение выводит карточки пользователей, содержащие имя, фотографию и кнопку для перехода к детальной информации, а также обеспечивает переход на страницу профиля выбранного участника.

2. **Используемые технологии**  
- JavaScript  
- XMLHttpRequest для обращения к VK API  
- Модульная архитектура: модули (ajax, urls, consts) + компоненты (UserCard, Filter, UserPage)  

3. **Функциональность приложения**  
В рамках лабораторной работы №4 реализованы следующие функции:  

- Загрузка списка участников заданной группы VK через метод `groups.getMembers` и их отображение на главной странице в виде карточек.  
- Отрисовка карточек с именем и аватаром пользователя; при нажатии кнопки «Посмотреть профиль» осуществляется переход на страницу профиля.  
- Загрузка детальной информации о пользователе (имя, фамилия, фотография, город, дата рождения, идентификатор).  
- Фильтрация списка участников по категориям на главной странице (компонент Filter).  
- Разделение на уровни: модуль `ajax` инкапсулирует работу с XMLHttpRequest, модуль `urls` формирует URL‑адреса для запросов к VK API, а компоненты отвечают за рендеринг интерфейса.  
- Кнопка возврата на главную страницу со страницы профиля пользователя.  

4. **Инструкция по запуску**  
**Требования**  
- Современный браузер (Chrome, Firefox)  
- Доступ к интернету для подключения к VK API  
- Расширение для обхода CORS в браузере  

5. **Использование приложения**  
- **Главная страница:** Просмотр списка участников группы с возможностью применения фильтров.  
- **Фильтрация:** Ввод текста в поле фильтра для поиска участников по имени.  
- **Профиль:** Нажатие кнопки «Посмотреть профиль» на карточке пользователя для открытия подробной информации.  
- **Возврат:** Использование кнопки «← Вернуться» для возврата на главную страницу.  

6. **Технические особенности**  
- **XMLHttpRequest vs Fetch**  
В данной работе применяется классический XMLHttpRequest для демонстрации традиционного подхода к выполнению асинхронных запросов. Более современный метод с использованием Fetch API будет рассмотрен в лабораторной работе №5.  

- **CORS и безопасность**  
  - VK API требует корректных CORS‑заголовков при запросах с фронтенда.  
  - Для локальной разработки используется расширение браузера для обхода CORS.  
  - В производственной среде необходимо применение бэкенд‑прокси или серверного API.  

7. **Заключение**  
Лабораторная работа №4 иллюстрирует практическое применение XMLHttpRequest для взаимодействия с внешними API, организацию кода в виде модулей и компонентов, а также ключевые принципы построения одностраничного веб‑приложения на JavaScript.
