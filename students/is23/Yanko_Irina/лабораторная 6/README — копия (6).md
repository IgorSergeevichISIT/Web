**Лабораторная работа №6: Веб‑приложение «Участники VK‑группы» (Node.js + Fetch API + BFF)**

1. **Общая характеристика работы**
В ходе выполнения лабораторной работы №6 было разработано клиент‑серверное веб‑приложение для управления списком участников группы социальной сети VK. Архитектура решения построена по паттерну **Backend For Frontend (BFF)**, где сервер на **Node.js** выполняет роль промежуточного звена между пользовательским интерфейсом и **VK API**. На стороне клиента для взаимодействия с сервером используется современный **Fetch API**, что обеспечивает удобную и эффективную работу с асинхронными запросами.

2. **Реализованная функциональность**
Ключевые возможности приложения:

- Загрузка данных об участниках группы через **VK API** с последующим кэшированием в локальном файле `users.json`

- Отображение списка пользователей в виде карточек, содержащих имя, фотографию и статус

- Добавление новых участников через всплывающее модальное окно (AddUserModal)

- Редактирование информации о пользователе (имя, статус, город, дата рождения)

- Удаление пользователей из списка

- Просмотр расширенной информации о пользователе на отдельной странице профиля

- Фильтрация списка по имени и статусу

**REST API эндпоинты, предоставляемые сервером:**

- `GET /api/health` — проверка работоспособности сервера

- `GET /api/users` — получение списка всех пользователей (с поддержкой фильтрации)

- `GET /api/users/:id` — получение данных конкретного пользователя

- `POST /api/users` — добавление нового пользователя

- `PUT /api/users/:id` — обновление информации о пользователе

- `DELETE /api/users/:id` — удаление пользователя


3. **Руководство по запуску проекта**
Системные требования:

- Node.js версии 14 или выше

- npm (поставляется вместе с Node.js)

- Доступ в Интернет для обращения к VK API

Последовательность запуска:

Установите зависимости проекта:

```bash
npm install
``` 
Запустите серверное приложение:

```bash
node server/server.js
```
Для автоматической перезагрузки сервера при изменениях в коде рекомендуется использовать nodemon:

```bash
npx nodemon server/server.j
```
Откройте браузер и перейдите по адресу:


`http://localhost:3000`

4. **Описание работы с приложением**
На главной странице:

- Отображается список участников группы в виде карточек

- Доступна строка поиска для фильтрации по имени

- Кнопка «Добавить пользователя» открывает модальное окно для ввода данных

- Клик по карточке пользователя ведёт на страницу с его полным профилем

В модальном окне добавления:

- Необходимо заполнить поля: имя, фамилия, статус, город, дата рождения

- Для сохранения нового пользователя используется кнопка «Добавить»

На странице профиля пользователя:

- Представлена полная информация о пользователе

- Доступны формы для редактирования данных

- Присутствует кнопка удаления пользователя из списка

- Кнопка «Вернуться» осуществляет переход обратно на главную страницу

**Архитектурный подход BFF (Backend For Frontend)**
Применение данного подхода обусловлено следующими преимуществами:

Защита конфиденциальных данных: токен доступа к VK API остаётся на сервере и не раскрывается клиенту

Адаптация данных: сервер преобразует ответ внешнего API в формат, удобный для отображения во фронтенде

Оптимизация запросов: кэширование результатов снижает количество обращений к VK API и повышает скорость работы

Упрощение поддержки: изменения во внешнем API не требуют модификации клиентского кода
