**Лабораторная работа 3: Разработка веб-приложения с использованием чистого JavaScript и Bootstrap**  
**Цель работы:** Освоение базовых инструментов фронтенд-разработки, включая Node.js, npm, Bootstrap 5 и нативные ES-модули. Реализация двухстраничного веб-приложения с динамической генерацией контента без использования фреймворков. В рамках работы применяются компонентный подход, манипуляции с DOM, обработка событий и система уведомлений.

---

### **Структура проекта**
- **css/** — кастомные стили:  
  `style.css` — стили для карточек, уведомлений и страниц.
- **images/** — изображения для карточек:  
  `image1.jpg`, `image2.jpg`, `image3.jpg` (иллюстрации финансовых акций/кошек).
- **components/** — переиспользуемые UI-компоненты:  
  - `product-card/index.js` (или `cat-card/index.js`) — карточка элемента на главной странице с кнопками «Подробнее», «Связаться», «Удалить».  
  - `product/index.js` (или `cat-profile/index.js`) — детальная карточка на странице элемента.  
  - `back-button/index.js` — кнопка возврата на главную страницу.  
  - `toast/index.js` (или `alert/index.js`) — компонент всплывающих уведомлений (Toasts/Alerts).
- **pages/** — логика страниц:  
  - `main/index.js` — главная страница: отображение списка элементов, добавление/удаление карточек, обработка событий.  
  - `product/index.js` (или `cat/index.js`) — страница детального просмотра выбранного элемента.
- **index.html** — точка входа. Подключает Bootstrap, кастомные стили и главный скрипт.
- **main.js** — инициализация приложения, хранение состояния (список карточек), управление навигацией между страницами.
- **package.json, package-lock.json** — метаданные и зависимости проекта (Bootstrap).
- **.gitignore** — исключение служебных файлов из Git (например, `node_modules/`).
- **README.md** — документация по проекту.

Проект использует нативные ES-модули (`import/export`) и не требует сборки (например, через Webpack). Весь код выполняется в браузере.

---

### **Функциональность**
1. **Двухстраничное приложение:** главная страница (список элементов) и страница детального просмотра.
2. **Динамическая генерация карточек** на основе данных (без хардкода в HTML).
3. **Добавление карточек по циклу:** при нажатии на кнопку «+ Добавить акцию»/«Добавить кошку» элементы добавляются последовательно (1 → 2 → 3 → 1 → ...).
4. **Удаление карточек** — каждая карточка имеет кнопку «Удалить» (работает только на главной странице).
5. **Всплывающие уведомления** (Toasts/Alerts) — отображаются при действиях пользователя (например, «Связаться», «Запросить анализ», «Добавить в избранное»).
6. **Навигация:**  
   - Переход между страницами без потери состояния (добавленные/удалённые карточки сохраняются).  
   - Кнопка «Домой»/«Назад» для возврата на главную страницу.
7. **Сброс состояния при перезагрузке** — после обновления страницы отображаются только исходные карточки.
8. **Адаптивная вёрстка** с использованием Bootstrap 5.
9. **Чистая архитектура:** разделение на страницы и компоненты, инкапсуляция логики.

### **Инструкция по запуску**
**Требования:**
- Установленный Node.js (для работы с npm).
- Расширение Live Server в VS Code (или любой локальный веб-сервер).

**Шаги:**
1. Установите зависимости:  
   ```bash
   npm install
   ```
   Команда установит Bootstrap в папку `node_modules/`.
2. Подготовьте изображения:  
   Убедитесь, что в папке `images/` находятся файлы `image1.jpg`, `image2.jpg`, `image3.jpg`. При отсутствии изображений можно временно заменить пути в `main.js` на URL-плейсхолдеры.
3. Запустите приложение:  
   - Откройте проект в VS Code.  
   - Нажмите ПКМ на файле `index.html` → **Open with Live Server** (или используйте другой локальный сервер).  
   - Приложение откроется в браузере (обычно на `http://127.0.0.1:5500`).

**Важно:** Приложение не требует серверной части. Вся логика выполняется в браузере.

### **Технические особенности**
- Все переходы между страницами реализованы через перерисовку DOM (без загрузки отдельных HTML-файлов).
- Состояние приложения (список карточек) хранится в `main.js` и сохраняется между переходами, но сбрасывается при перезагрузке страницы.
- Для генерации уникальных идентификаторов карточек используется временная метка (`Date.now()`).
- Всплывающие уведомления реализованы с использованием Bootstrap-стилей и автоматически скрываются через 5 секунд.
- Данные элементов (например, финансовые акции или кошки) хранятся в виде массива объектов в модуле главной страницы (`pages/main/index.js`).
