**Лабораторная работа №5: Веб‑приложение «VK группа» (Fetch API)**


1. Краткое описание
Лабораторная работа №5 представляет собой развитие предыдущей работы (ЛР №4), в которой был осуществлен переход с устаревшего XMLHttpRequest на современный Fetch API. Также в проекте были применены принципы проектирования S.O.L.I.D. для создания чистого, гибкого и расширяемого кода.

2. Используемые технологии
JavaScript (стандарт ES6+)
Fetch API для выполнения запросов к VK API (взамен XMLHttpRequest)
Модульная архитектура приложения, включающая:
папку modules/ — для служебных модулей (urls, consts)
папку components/ — для UI-компонентов (Filter, UserCard, UserPage)
файл main.js — в котором находится основная логика приложения

4. Функциональность приложения
Загрузка перечня участников группы VK с использованием Fetch API.
Отображение карточек пользователей, содержащих фотографию, имя и кнопку для перехода в профиль.
Фильтрация списка участников по заданным категориям (друзья, руководители и другие).
Просмотр детальной информации профиля выбранного пользователя (дата рождения, город и другие данные).
Возможность возврата на главную страницу со списком участников.

5. Инструкция по запуску
Требования
Современный веб-браузер (например, Chrome, Firefox).
Наличие подключения к интернету для обращения к VK API.
Установленное CORS-расширение для браузера либо использование dev-сервера.


В ходе выполнения лабораторной работы №5 были продемонстрированы:

Применение Fetch API на практике для взаимодействия с внешним API.
Использование архитектурных принципов S.O.L.I.D. для обеспечения гибкости и удобства поддержки кода.
Организация модульной структуры с четким разделением зон ответственности.
Использование callback-функций для реализации инверсии зависимостей.
